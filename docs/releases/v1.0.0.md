# v1.0.0

First polished EE508 course-ready release.

## Highlights

- structured CARC setup flow for requesting GPU nodes and starting a CUDA-ready shell
- working CUDA smoke test for validating compilation and execution on a compute node
- profiling workflows for Nsight Systems and Nsight Compute
- EBS-aware `ncu` workflow with node-status guidance for `e23-02`
- student-oriented CUDA examples covering threads, blocks, grid-stride loops, vector add, and reduction
- cleaned documentation with portable repository paths and parameterized CARC account handling

## Included Areas

- `setup/` for Slurm allocation, environment setup, and smoke testing
- `profiling/` for `nsys` and `ncu` workflows
- `cuda/` for teaching examples and simple benchmarks

## Notes

- `run_profile.sh` has been verified working in this environment
- `run_ncu.sh` depends on EBS and GPU performance counter availability on the target node
- scripts now use `CARC_ACCOUNT` or `${USER}_1190` instead of a hardcoded account value

`v1.0.0 - First polished EE508 course-ready release`

## Suggested GitHub Release Description

This release packages the initial polished version of `carcusc` for EE508 students working on CUDA at USC CARC.

It includes:

- one-command startup with `./setup/start_cuda.sh`
- a CUDA smoke test with `bash ./setup/test_cuda.sh`
- profiling helpers for `nsys` and `ncu`
- small, focused CUDA teaching examples
- documentation cleaned for portability and student onboarding
